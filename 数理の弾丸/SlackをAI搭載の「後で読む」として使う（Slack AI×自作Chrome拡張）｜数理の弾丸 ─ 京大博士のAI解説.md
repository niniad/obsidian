## SlackをAI搭載の「後で読む」として使う（Slack AI×自作Chrome拡張）

参加中

Xのポストや技術記事を見かけたとき、「いま手を止めて読みたくはないが、  
後で読みたい」と思うことが頻繁にあります。「後で読む」の保管・記録・読込の体験を良くするためにやっていることを紹介します。

この記事は全て人間が書いています。有料エリアには、この記事の内容を実践するための手順やプロンプトを置いています。

＊　＊　＊

Slackで、と書いてはいますが、Slackである必然性はなく、以下の条件を満たす「後で見る」が欲しいので作ったという話です。

- **デバイス間で直ちに共有されること**  
	PCでの作業がモバイルにも反映される、など
- **情報をカテゴリに分けて整理できること**  
	Xのブックマーク等はこれができず辛い
- **検索がしやすいこと**  
	これも同様にXのブックマークだと厳しい
- **ブラウザ上の少ない操作で記録できること**  
	後で見るに追加する手間を、ボタンを2〜3回押すくらいに留めたい
- **AIとの連携が可能なこと**  
	どんな情報があるかAIに聞きたい（後で見るが増えると忘れる・見失う）

## 大抵の情報をXで集めている

良さそうなツール・技術記事・論文などの情報の大部分をXから収集しています。Xにはブックマーク機能があり、気になったポストをそこに置いておくことができます。

しかしXのブックマークには、私にとって大きなデメリットがいくつかあります。クリティカルな順に書くと：

- Xのポストしか記録できない（当然ですが）
- 全ポストが1つのシーケンスとして蓄積され、カテゴリごとに整理するといったことができない
- 検索性が良くない
- AIとの距離が遠い

という感じです。先に書いた条件でいうと「デバイス間で直ちに共有されること」と「ブラウザ上の少ない操作で記録できること」しか満たせていません。

そこで作りたいと思ったのが下記を満たす保管庫とそれを取り巻く仕組みです。

- デバイス間で直ちに共有される
- 情報をカテゴリに分けて整理できる
- 検索がしやすい
- ブラウザ上の少ない操作で記録できる
- AIとの連携が可能

## Slackがベストマッチだった

これらを満たす保管庫として私はSlackを採用しています。Slackはチャンネルという境界で情報の区分けをすることができ、本来は話題・プロジェクトごとにチャンネルを分けてそこで会話する、という使い方をします。

私は参加者が自分だけのSlackワークスペースを作り、トピックごとにチャンネルを分けて、そこに気になったサイトやポストのリンクを投げています。

![画像](https://assets.st-note.com/img/1771590202-rFYl20ovzcZOV5s6ey8iatgj.png?width=1200)

サイドバーに並んでいるのがチャンネル、メイン領域にチャンネルの内容

Slackの有料プランはSlack AI機能が利用可能で、チャンネルの要約や、Slack上の投稿に関する質問とかができます。Slack AIでなくても、Claudeなどのチャットボットをアプリとして導入することができます。あるいはSlack MCPを使ってClaude Code等とSlackを連携させることも可能です。

シンプルにキーワード検索もできます。Xブクマはこれができずしんどい。

![画像](https://assets.st-note.com/img/1771590247-46JnbSp3V0XyQvqFAldHuGMg.png?width=1200)

個人的に気に入っていることとして、メッセージアプリであるが故に、「後で読む」にリプライツリーをぶら下げることができます。 **ブックマークと自分の思考ログの融合** という感じで使えます。

![画像](https://assets.st-note.com/img/1771590284-9fJjuMQbCyx02YaAU5eKmncp.png?width=1200)

ブクマに思考やメモをぶら下げるイメージ

また、デスクトップ版とモバイル版の両方があり、それぞれの体験に大きな差がないのもSlackの推しポイントです。

これで、要件のうち4つは実現できました。

- ○ デバイス間で直ちに共有される
- ○ 情報をカテゴリに分けて整理できる
- ○ 検索がしやすい
- ○ AIとの連携が可能
- × ブラウザ上の少ない操作で記録できる

あとは、 **サイトをSlackに投げる** という作業を最小限の手間でできたら最高です。素朴にやると、ブラウザからSlackに画面切り替えが発生して煩雑です。そこで、 **開いているサイトのURLをSlackに投稿するChrome拡張** を作成しました。

## 自作Chrome拡張

機能はシンプルで、サイト上でショートカットを押したら下のようなポップアップを表示して、選んだチャンネルに投稿できるというものです。

![画像](https://assets.st-note.com/img/1771587629-ae3F5pUqT97wKbgXiBRs6Smo.png?width=1200)

ブクマ+SlackでBUKUMASと適当に名付けた

これで、ブラウザから移動せずに後で見るを記録できるようになりました。

ObsidianのWeb Clipperを使うと、ここまでの内容＋サイトの内容をマークダウンで保存、という形で実現できます。Obsidianユーザにはおすすめですが、私はObsidianあまり使っておらず、普段Slackを見ている時間が多いのでこうしています。また、Obsidian Web ClipperのChrome拡張はサイト情報をObsidianに記録するたびにObsidianアプリに飛ばされるので、そこの体験面でも却下しました。

最後に、上記を実現するための手順と、Chrome拡張を作るためのプロンプトをご紹介します。ここからは有料エリアにしておきます。

## 実現手順

実際に上記を実現する手順をご紹介します。

### 1\. Slack ワークスペースを作成する

まず後で見るを置いておく用のSlackワークスペースを作成します。このワークスペースにチャンネルを立てて、そこに投稿をポイポイ投げる感じです。

### 2\. Slack App を作成する

SlackにはSlack Appという機能があり、ワークスペースにインストールすることでBotのように使うことができます。

下記からAppを作成します。「Create New App」からの「From scratch」を選択します。

アプリ名は適当に設定してください。また、連携先のワークスペースは先ほど作ったものを選択します。

この後に作る拡張機能がこのAppを制御することで、Slack自動投稿を実現します。

### 3\. Bot Token Scopesの設定

Appがワークスペースに対して持つ権限を設定します。

Appの設定画面にて、左サイドバーのOAuth&Permissionsを押します。

![画像](https://assets.st-note.com/img/1771588553-BgcFswSWoT10n7ECyHmrMK8Q.png?width=1200)

下にスクロールすると、Scopesという設定項目があると思います。ここで「Add an OAuth Scope」をクリックして下記の権限を付与します。

- channels:read  
	パブリックチャンネル一覧の取得
- groups:read  
	プライベートチャンネル一覧の取得
- channels:join  
	Botのチャンネル参加
- channels:manage  
	チャンネル新規作成・ユーザー招待
- chat:write  
	メッセージの投稿

### 4\. ワークスペースへのインストール

同じ「OAuth & Permissions」ページ上部の「Install to Workspace」をクリックして、Appをワークスペースにインストールします。

そして、表示される「Bot User OAuth Token」(xoxb-... で始まる文字列) をコピーしておいてください。

これでApp側の準備は完了です。次に拡張機能を作成します。

### 5\. Chrome拡張の作成

適当な空ディレクトリを作成し、そこに作成します。Claude Codeなどに下記のプロンプトを投げればそれでいけると思います。拡張機能名をBUKUMASにしていますが、適当に変えて大丈夫です。

```ruby
# BUKUMAS - Slack リンク共有 Chrome 拡張機能

## 背景

日常的に X（Twitter）や Web サイトのリンクを Slack チャンネルに手動で投稿している。このワークフローを Chrome 拡張機能で自動化する。

## 基本フロー
1. 拡張機能を起動
2. チャンネルを選択
3. リンクを Slack に投稿

## ポップアップのレイアウト（上から順に）

1. タイトル「BUKUMAS」
2. ページ情報（現在ページのタイトルと URL を自動表示）
3. チャンネル選択ドロップダウン（既存チャンネル一覧 + 末尾に「+ 新規チャンネル」選択肢）
4. チャンネル名入力フィールド（「+ 新規チャンネル」選択時のみ表示）
5. コメント入力欄（任意）
6. 「投稿する」ボタン
7. ステータスメッセージ（成功・エラー表示用）

Bot Token 未設定時は、メインコンテンツを非表示にし、設定画面へのリンク付き誘導メッセージを表示する。

## UI 要件

- UI メッセージは日本語で表示する
- チャンネル一覧はアルファベット順でソートし、名前の先頭に \`#\` を付与する
- 公開チャンネルとプライベートチャンネルの両方を表示し、アーカイブ済みは除外する
- 前回選択したチャンネルを起動時に自動選択する
- 「+ 新規チャンネル」選択時はチャンネル名入力フィールドにフォーカスを移す
- チャンネル新規作成時、設定済みのユーザー ID を自動招待する（招待失敗時も投稿は続行）
- Bot が未参加のチャンネルには投稿前に自動参加する
- 投稿メッセージのフォーマット: コメント（任意）+ タイトル + URL の改行区切り。タイトルが URL と同一の場合はタイトルを省略する
- リンクプレビューを有効にする（\`unfurl_links: true\`）
- 投稿成功後、1 秒後にポップアップを自動クローズする
- Bot Token はパスワードフィールド（マスク表示）で入力する
- Options Page で Token 未入力のまま保存しようとした場合はエラーメッセージを表示する

## Options Page のレイアウト（上から順に）

1. タイトル「BUKUMAS 設定」
2. Slack Bot Token 入力フィールド（パスワード型）
3. Slack ユーザー ID 入力フィールド（チャンネル新規作成時の自動招待用）
4. 「保存」ボタン
5. ステータスメッセージ

## 技術仕様

- Manifest V3 Chrome 拡張（Popup UI / Options Page の 2 構成）
- Chrome permissions: \`activeTab\`, \`storage\`
- host_permissions: \`https://slack.com/api/*\`
- 認証: Slack Bot Token を \`chrome.storage.local\` に保存する（個人利用前提。OAuth は不要）
- ポップアップが現在ページの URL とタイトルを自動取得する
- バニラ HTML/CSS/JS、ビルドツールなし

## 使用する Slack API

- \`conversations.list\`: チャンネル一覧取得（カーソルベースのページネーションで全件取得）
- \`conversations.create\`: チャンネル新規作成
- \`conversations.invite\`: ユーザーをチャンネルに招待
- \`conversations.join\`: Bot をチャンネルに参加させる
- \`chat.postMessage\`: メッセージ投稿

## 必要な Slack Bot Token Scopes

- \`channels:read\` - 公開チャンネル一覧の取得
- \`groups:read\` - プライベートチャンネル一覧の取得
- \`channels:join\` - Bot の公開チャンネルへの自動参加
- \`channels:manage\` - チャンネルの作成・ユーザー招待
- \`chat:write\` - メッセージの投稿
```

### 6\. Chrome拡張の導入

自作拡張機能を導入する一般的な手順を踏みます。

1. Chromeで chrome://extensions を開く
2. 右上の「デベロッパーモード」をオンにする
3. パッケージ化されていない拡張機能を読み込む をクリック
4. 拡張機能を作成したフォルダを選択

これで、作成した拡張機能が一覧に追加されます。

### 7\. 拡張機能とSlack Appの連携

ツールバーの拡張機能のアイコンを右クリックし、「オプション」へ飛びます。4で取得したBot User OAuth Tokenと、あなたのSlackユーザーIDを入力するフォームが表示されると思います。

![画像](https://assets.st-note.com/img/1771589137-OCQjIyS1XHAxq3v72ZWaJwuR.png?width=1200)

イメージこんな感じ

これらを設定することで、拡張機能がAppを制御できます。

個人利用の範囲なら実害は少ないと判断して良しとしていますが、このやり方だとBotトークンがChrome Local Storageに平文保存されるため、悪意をもった拡張機能がこれを読む可能性はゼロではなく、また、PCを共用している人がいればその人が悪用することも可能な状態です。

拡張機能の管理には十分に注意し、完全に個人用のPCでのみ、自己責任で導入してください。それでもやはり気になる場合はWeb Cryptoを使って暗号化するなどを検討してください。

### 8\. キーボードショートカットの設定

ショートカットを設定しておくと良いと思います。私はCmd＋Bで拡張機能が有効化されるようにしています。

![画像](https://assets.st-note.com/img/1771589229-kOKMiupUCrz08IbGqB1D9xTA.png?width=1200)

関係ないですが、サイトのテキストをマークダウン形式でコピーする拡張機能も自作して使っています

手順は以上です。何か不具合があったらAIと相談しながら解消していただけると幸いです・・・！
