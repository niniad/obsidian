## 秒で導入できる、Claude Codeを快適にするための工夫

こんにちは。弾丸です。お陰様でつい最近、YouTubeの登録者が10,000人を超えました。今後ともよろしくお願いいたします。

![画像](https://assets.st-note.com/production/uploads/images/255699925/picture_pc_95798833ea5e93218577b64c9bb3cd33.png?width=1200)

この記事では、Claude Code（をはじめとするコーディングエージェント）での日々の作業を少しだけ快適にするために私がやっていることを共有します。

特定のユースケースでの改善というより、あらゆる場面での体験が少し良くなる（小さなイライラを減らす）ためのものと思ってみていただければと思います。

また、これ良かったという話と同じかそれ以上に、「これは微妙だったからやめた」みたいな情報も意味があると思うので、それはそれで記事にしたいと思っています。

## 要点

先に列挙しておきます。もちろん他にもいろいろやっているのですが、特に「小さなイライラを減らす」という点で有用なものを、導入が楽なものから挙げています。

- マークダウン太字と斜体の禁止
- お前の知識など知らんプロンプティング
- 消費コンテキストの常時可視化
- claude-memによるセッションを跨いだ履歴保持・参照
- portlessによるポート競合との決別

では順に、簡単に説明しておきます。

## マークダウン太字と斜体の禁止

Claudeに限らず、LLMたちはやたらと「\*\*」で強調してきます。とても邪魔で嫌なので、グローバルのCLAUDE.mdに以下を書いています。

```
**による太字や*での斜体など余計な装飾を厳禁とします
```

これだけで相当見やすくなります。

## お前の知識など知らんプロンプティング

エージェントとの作業の大部分は、計画を含む数多の壁打ちです。Web上の情報を走査して、解決策の案出しをしたり、自分がインプットするためのドキュメントを作らせたりといったことを多々行います。

基本的に、私はエージェントが元々持っている知識を信用していません。最新LLMの例としてGPT-4oとかGemini 1.5を提案してきたりするので、私が詳しくない領域の知識も大体そんな感じで古いものが混じってるのだろうと思っています。

モデル自身の知識をベースに会話するより、むしろ膨大な調査結果をこちらの要求に合わせて文書化するエンコーダとして使う方が体験が良いです（このエンコーディング能力がめちゃくちゃ高い）。

そういうわけで以下の「お前の知識など知らんプロンプト」をグローバルのCLAUDE.mdに書いています。

```
私はあなたの持っている知識に興味はありません。私が何か質問をしたとき、それが作業中のリポジトリについてのものでない限り、原則として入念なWebサーチの上で回答してください。
```

コンテキストの消費は大きくなりやすいですが、それに見合う快適さがあります。エージェントが自分の知識に基づいて回答してきたのに対して「Webサーチして根拠をもとに回答してください」と依頼する、みたいな無駄なやり取りを減らして、本来必要なことにコンテキストを使うという感じです。

## 消費コンテキストの常時可視化

Claude Codeは標準機能でコンテキストウィンドウの消費量を確認できるので、これを使ってステータスラインにコンテキスト残量が表示されるようにしています。

![画像](https://assets.st-note.com/production/uploads/images/255693572/picture_pc_645789f67b3a5a2f399fa7f79988fd60.png?width=1200)

めちゃくちゃ良いです。このTipsはXのポストで見かけて導入した気がするのですが見つけられず…。

下記をClaude Codeに指示するだけでいけます。

```javascript
/statusline {モデル名}  {作業ディレクトリ}  [{プログレスバー}] {使用率}%

この形式で表示して。バーは20文字幅で"#"と"-"を使用。ホームディレクトリは~に短縮。70%未満は緑、70-89%は黄、90%以上は赤。
```

ステータスラインをオシャレにするツールとかもあったりするのですが、私個人的にはそこまでリッチなものは必要ないかなという感じです。

## claude-mem

claude-memという、スキル+メモリハーネス的なツールを導入しています。有名かも。

[**GitHub - thedotmack/claude-mem: A Claude Code plugin that automatically captures everything Claude does during your coding sessions, compresses it with AI (using Claude's agent-sdk), and injects relevant context back into future sessions.** *A Claude Code plugin that automatically captures everything C* *github.com*](https://github.com/thedotmack/claude-mem)

導入すると、/mem-searchで別セッションの会話も含めて検索が可能になり、セッションを跨いだコンテキストの伝承がある程度可能になります。

大まかな仕組みは、記憶を記録する常駐サーバが立ち上がり、それがHookとして走ることで逐一やり取りが記録されるという感じです。

別セッションの会話を参照させたいときにとても便利なのと、個人的には内省の道具としてとても良いです（下記）。

```
/mem-search

ここしばらくのやりとりを参照して、私について〇〇の観点で分析してください。
```

〇〇には、例えば「どんなときに不満を覚えるか」とか、「AI駆動開発をよくするために暗黙的にやっている工夫はあるか」とか、「ミスにつながりやすい行動パターンはあるか」とかが入るイメージです。

これを使って、私は「細かい言い回しにこだわりすぎ」とか、「同時に複数の作業依頼を出しがち」とか、「適切なタイミングでレビューしてないことが多い」などなど、私自身の改善につながるヒントをたくさん得ています。

## portlessによるポート競合との決別

最後はportlessです。これも有名かも。

[**GitHub - vercel-labs/portless: Replace port numbers with stable, named.localhost URLs. For humans and agents.** *Replace port numbers with stable, named.localhost URLs. For* *github.com*](https://github.com/vercel-labs/portless)

任意のローカルサーバーを、{任意のプレフィックス}.localhost:1355で起動できるというやつです。

AI駆動になってから、複数の（Web）アプリを並行で開発するのが常態となり、ポートの競合がとんでもなく発生します。portlessはサーバーをポートではなくプレフィックスで区別できるようにしてくれるので、この問題がほぼなくなります。

私はportlessのプレフィックスを{リポジトリ名}-{ブランチ名}とすることで、名前の衝突が起こらないようにしています。おすすめです。
