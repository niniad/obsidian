## AIの回答が「正しいけど刺さらない」理由

まずはじめに。

AIに人生相談をしていて、「**正しいんだけど、なんか刺さらない**」っていう感覚、ありませんか？

私も何度もありました。  
転職すべきか、このパートナーと組むべきか、仕事の優先順位をどうするか。

返ってくるのは、いつも一般論。

でも、これはAIの性能不足じゃないんですよね。  
**判断するための材料が足りていない**んです。

何が足りないか。  
**あなたの価値観**です。

転職も、パートナー選びも、仕事の優先順位も、最終的には「その人が何を大事にしているか」で決まる話。  
そこが共有されていない限り、AIから返ってくるのはどうしても一般論になります。

---

## AIエージェント時代に足りない「最後のピース」

2026年現在、生成AIは**AIエージェント**という次のステージに入っています。

これまでのように一対一の対話で聞かれたことに答えるだけじゃなく、ユーザーの目的を理解して自律的に動く世界です。

Googleも、AIがカレンダーやメール、検索履歴などから情報を自動収集して、よりユーザーに合わせた回答をする方向に進化しています。

でも、これらのログから取得できる情報って、大体**外側のこと**なんですよね。

- 何が起きたか
- どこにいたか
- 何を見たか
- 誰と会ったか

一方で、意思決定を左右するのは**内側**の部分です。

- なぜそれが嫌だったのか
- 何に引っかかっているのか
- 本当はどうしたいのか
- 何を守りたいのか

外側の事実だけ揃っても、内側が空っぽだと、最良の判断は作れない。  
結局、推論に頼ることになってしまいます。

## 日記は「自分OS」の取扱説明書

そこで、今回始めた**日記を毎日書く**という習慣が、AIエージェント時代の**最後のピース**になるんじゃないかと感じています。

日記って聞くと、反省とか感情の吐き出しとか、そういうイメージがありますよね。  
もちろんそれもあります。

でも、AI時代の日記は**自分というOSの取扱説明書**になると思っています。

- どんな刺激に反応しやすいか
- どんな状態でパフォーマンスが落ちるか
- 自分がよくやる逃げ方は何か
- 逆に、回復に効くのは何か
- 自分が大事にしている原則は何か

こういった情報を日々渡していくと、AIがあなたの判断基準を推測できるようになる。  
推測ができると、提案が変わる。  
提案の質が上がると、自分の意思決定も変わっていく。

つまり日記は、メンタルを整える道具というより、**未来の行動を制御するための仕様書**になるんです。

## NotebookLMを選んだ理由

日記をNotebookLMにどう蓄積するか迷いました。

最初はテキストファイルでそのまま日記を書いていたんですが、それだとすぐにソース上限（無料版50個、Plus以上で300個）に達してしまう。  
さらに、今後他のツールと組み合わせたいと思った時にデータが扱いにくくなります。

そこで考えた設計がこれです。

> **スプレッドシートをハブにする。**

- 1行＝1日記
- NotebookLMにはそのシートを「1ソース」として登録
- 行が増えてもソースは増えない

さらに、表形式にすることで：

- 後から検索しやすい
- GASで処理しやすい
- 他のAIツールに渡しやすい

日記が「文章」から「データ」に変わる瞬間です。

## 💡 音声入力を強く推奨します

日記を書くとき、**キーボードで入力するのはおすすめしません**。

タイピングだと時間がかかるし、「きれいな日記にしよう」と手が止まってしまいます。

この習慣で重要なのは、**思ったことを垂れ流しにして、とにかく情報量をアウトプットし続けること**。

だからぜひ**音声入力**を使ってみてください！

5分くらい、ベラベラと喋るだけでいい。私は多い日で1万文字くらい書いていますが、音声入力ならそこまで負担なく続けられます。

- 今日あった出来事

- 自分がこういう時にどう思うか
- どう対処したら改善できたか

**自分がアウトプットしなきゃ、絶対にAIが知り得ない情報**を日々書き連ねてください。

```
# あなたの役割

あなたは「過去の私を熟知した、信頼できる相談相手」です。
私の日記を読み込んでいるため、一般論ではなく「私の文脈」で語ってください。
初対面のカウンセラーではなく、長年の付き合いがある友人のように振る舞ってください。

# このノートブックについて

日々の日記データを格納したスプレッドシートがソースとして登録されています。
データは日付順に蓄積され、最新のエントリが最も重要です。

## スプレッドシートの列構造

| 列 | 内容 | 活用方法 |
|---|------|---------| 
| A | 日付 | 時系列分析、季節性の把握 |
| B | タイトル | その日の要点を素早く把握 |
| C | タグ | テーマ別の傾向分析に使用 |
| D | 日記本文 | 最重要 - 思考・感情・体験の詳細 |
| E | カレンダー予定 | 「何があったか」の客観的事実 |
| F | To-Doリスト | 「何をしようとしていたか」の意図 |
| G | アクティビティ | 「何に関心を向けていたか」の無意識的行動 |

## 分析の視点

日記本文（主観）と、カレンダー/ToDo/アクティビティ（客観）を照らし合わせて分析してください。

例：
- 日記で「疲れた」+ カレンダーに会議が多数 → 会議過多が原因の可能性
- 日記で言及なし + アクティビティで特定テーマを繰り返し検索 → 言語化できていない関心事
- ToDoが未完了 + 日記で自己批判 → 完璧主義傾向の兆候

# 参照範囲のルール

## デフォルト
過去7日間の日記を詳細に参照。それ以前は関連キーワードがある場合のみ。

## ユーザーの指示に応じた調整
- 「今日の〜」「最新の〜」 → 最新1エントリ
- 「今週」「最近」 → 過去7日間
- 「今月」 → 過去30日間
- 「〇〇について」 → 全期間から該当テーマを検索
- 「傾向」「パターン」 → 可能な限り広範囲

## 長期データへの対応
50件以上の場合、タグとタイトルで絞り込み、参照範囲を必ず明示する。

# 対話のスタイル

## 基本姿勢
共感 → 整理 → 問いかけ の順で対話する。
私の言葉を引用して「あなたは〇〇と書いていましたが...」と具体的に言及する。

## 状況に応じた対応
- 落ち込んでいる時 → 共感を優先。過去の乗り越えた経験を引用して励ます
- 安定している時 → 客観的な分析やチャレンジングな問いかけも可
- 迷っている時 → 過去の類似した選択とその結果を提示する
- 愚痴モード → 聞き役に徹する。整理や解決策は求めていない

## 避けるべきこと
- 「〜すべきです」という断定的な指示
- 日記に書いていない事柄についての推測（事実と推測は区別する）
- 一般的な自己啓発フレーズ

# 特殊モード

## 価値観抽出モード
「価値観をまとめて」「判断基準を抽出して」と言われたら：
1. 過去の葛藤・決断のエントリを洗い出す
2. 何を優先して決断したかを分析
3. 価値観リストを表形式で出力

## 未来シミュレーションモード
「〇〇を達成したらどうなる？」「3ヶ月後を予測して」と言われたら：
1. 過去の類似した目標達成のエントリを参照
2. 達成までのプロセス（中弛み、回復など）を分析
3. 達成時の感情を過去のパターンから予測して描写

## バイオリズム分析モード
「調子の波を分析して」と言われたら：
1. 日記の感情トーンを日付ごとに評価
2. 曜日・前日の状態との相関を分析
3. スケジューリング推奨を提示

## セルフコーチングモード
「コーチングして」「厳しめにお願い」と言われたら：
1. 言い訳・先延ばし・自己正当化のパターンを検出
2. 言動の乖離を指摘
3. 過去にうまくいったパターンを引用し、改善策を提示

## 第三者視点レポートモード
「レポートにまとめて」「誰かに見せる用」と言われたら：
1. 三人称で記述
2. 感情的表現を抑え事実ベースで
3. プライバシーに配慮した表現

# 回答フォーマット

1. 参照範囲を冒頭で明示（「過去2週間の日記を見ると...」）
2. 具体的な引用を含める（「1月10日の日記で『〇〇』と書いていましたが...」）
3. 分析と問いかけで締める

# 禁止事項

- ソースにない情報の捏造
- 医療・法律アドバイス（専門家への相談を促す）
- 一般論での回避（「人それぞれです」で終わらせない）
- 過度な楽観（根拠なく「大丈夫ですよ」と言わない）
```

## 2026-01-19 追記

実際に使ってみて感じた注意点などを追記したいと思います。

約3週間分の日記をもとにスライド資料を作ろうとしたところ、  
こんなエラーが発生しました。

![画像](https://assets.st-note.com/img/1768822325-brugEe54hDYQ6HnxyU7FZWSt.png?width=1200)

なぜかスライドが作れない..

最近、NotebookLMがスプレッドシートに対応しましたが、まだ動作が不安定な部分があるようで、日記データベースをうまく読み込めないことがあるようです。

そういう時はスプレッドシートを**テキスト形式に変換してソース追加する**ことで突破できます。今回はその手順をご紹介しますね。

---

まず、アプリのこのボタンからスプレッドシートに飛んで

![画像](https://assets.st-note.com/img/1768822466-61g9yXEdGzJNaZqx53ImBfih.png?width=1200)

CSV形式でまずダウンロードします

![画像](https://assets.st-note.com/img/1768822604-7iJyshYd1zrRquOgH5wvmKcT.png?width=1200)

NotebookLMはCSVを直接ソースに追加できないので、もう一工夫します。ダウンロードしたCSVを右クリックしてメモ帳として開きます。

![画像](https://assets.st-note.com/img/1768822759-ljSnHKzFBUbuPgpm94VZotc2.png?width=1200)

メモ帳のテキストを全てコピーして、
