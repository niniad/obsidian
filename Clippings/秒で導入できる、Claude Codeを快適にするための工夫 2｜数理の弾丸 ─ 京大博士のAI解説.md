## rewind（esc+esc）：会話やコード変更を無かったことにする

Claude Code標準の /rewind です。

### 解決する課題

AIにメッセージを送ったとき、「今の無かったことにしたい」という瞬間があると思います。例えば：

- 全角にし忘れて「soredeha,」みたいなのを送ってしまった
- 音声入力の文字起こしミスがあるまま送ってしまった
- 本題とズレたやりとりが発生してしまった

とかです。特に音声入力派の方は、音声入力からの即Enterで変な文字起こしのまま送ってしまうことは多いのではないかと思います（「issue」が「異臭」になってる、とか）。

### rewind

Claude Code上で /rewind あるいは esc 2連打で、特定のユーザーメッセージまで会話やファイル変更を巻き戻すことができます。

rewind をコマンドまたはショートカットでトリガーして、戻る先のメッセージを選択すると下記のような選択肢が出てきます。

![画像](https://assets.st-note.com/img/1773139837-G3TXcPFAOxVSkd2Ly78vRwZH.png?width=1200)

👆これ、めっちゃ便利

例えば「issue 13」と打ちたいところ、音声入力ミスで下記のように送ってしまったとします。

![画像](https://assets.st-note.com/img/1773139945-a6hJOne7dbpPusqAEZ3fig1y.png?width=1200)

こういうときに esc 2連打して、送信前の状態に戻すことができます。

![画像](https://assets.st-note.com/img/1773139996-pA60mwGcLW5uNzYxDoEdfvKR.png?width=1200)

## ディレクトリを跨いでセッションを引き継ぐ

ちょっとしたハックという感じです。

### 解決する課題

Claude Codeのセッションは、起動したディレクトリに紐づいています。例えば dir-a というディレクトリで起動したセッションを、途中から別のディレクトリ dir-b で使うことはできません。

困るのはどういうときかというと、例えばプロジェクトAでの調査や議論をもとにプロジェクトBで作業したいケースです。私の場合は、文献調査用のプロジェクトで積み上げたAIとのやりとりを、実験管理のプロジェクトに持ち越したい場面がありました。

ドキュメント化して新しいセッションでのClaudeに渡せば別に良いという場面もありますが、それまでの会話で積み上げた文脈が全部消えるのが地味に不便です。

### セッションの実体を考える

Claude Codeの各セッションでの履歴は、以下のパスにjsonlファイルとして保存されています。

```ruby
~/.claude/projects/{プロジェクトハッシュ}/{セッションID}.jsonl
```

プロジェクトハッシュは、作業ディレクトリの絶対パスのスラッシュをハイフンに置き換えたものです。例えば /Users/me/dir-a なら以下のようになります。

```ruby
~/.claude/projects/-Users-me-dir-a/3d946ed0-7075-4f5c-851e-c3a494276cf0.jsonl
```

Claude Code は「そのディレクトリに紐づくセッションかどうか」を、プロジェクトハッシュで識別しています。つまり、プロジェクトハッシュを他のディレクトリのものに書き換えれば、そのセッションは他のディレクトリと紐づくことになります。

### やり方

簡単なデモとして、~/tmp/dir-a から ~/tmp/dir-b への引き継ぎをやってみます。とりあえず ~/tmp/dir-a で Claude Code のセッションを立ち上げて、適当に質問します。

![画像](https://assets.st-note.com/img/1773141891-JcyRTjKi9AmLZ6OwE8xpNVDC.png?width=1200)

dir-a の CLAUDE.md には、めっちゃポジティブに回答するように指示を入れています。dir-b の CLAUDE.md には、めっちゃネガティブに回答するように指示を入れています。移行後に読まれる CLAUDE.md も切り替わるか、後で確認します。

すると ~/.claude/projects に下記のディレクトリが生えます（1favs は私の Mac のユーザー名です）。

![画像](https://assets.st-note.com/img/1773142566-zYdfa15kKHqXmW7BnG2IRe6U.png?width=1200)

dir-a のセッションを格納するディレクトリ

このディレクトリを複製して、"dir-a" を "dir-b" にリネームします。これで ~/tmp/dir-b のセッションとして認識されます。

![画像](https://assets.st-note.com/img/1773142796-n065iQWyVK2eYHguBDPbv3A4.png?width=1200)

フォルダを複製し、リネーム

そして ~/tmp/dir-b で Claude Code を立ち上げて /resume すると、dir-a で作成したセッションが表示されます🙌

![画像](https://assets.st-note.com/img/1773142654-qLEpolWTQ9NAZDw6kgFS7ihI.png)

※ /resume は過去に作成したセッションの再開です

先ほどの会話に続けて適当に何か入れてみます。

![画像](https://assets.st-note.com/img/1773143064-bBq6H2GRn74ywxKYVtAugakh.png?width=1200)

ネガティブ寄りですね

このように、元々セッションがあった dir-a ではなく移行後の dir-b に格納してある CLAUDE.md が読まれます。
